# 開発計画: cabaguide トップページ

## 1. 概要
- 目的: cabaguide トップページを新規開発し、トップキャストの訴求と主要導線の提供を行う。
- 成果物: Next.js ベースのトップページ UI、必要なコンポーネント、スタイルガイド。
- スコープ外: 詳細ページへの遷移先実装、フッター各リンクの遷移先実装。

## 2. 要件整理
### 機能
1. 2行構成のヘッダー (ブランドロゴ/スローガン、ホーム遷移)。
2. エリアから探すボタン (TOP) とその下の広告エリア (TOP)。
3. 10地域分のトップキャストカード列 (画像・名前・総フォロワー数・店舗名、各リンク設定)。
4. 広告エリア (BOTTOM) とフッター (8導線)。

### UI/表現
- フォント: 装飾的サンセリフ + 見出し太字/本文シンプル。
- カラー: 濃紺/黒ベース、ピンク・紫のグラデーション発光アクセント。
- 効果: グロー、ホバーで輝度アップ。
- 背景: ネオン粒子、曲線装飾。

## 3. 実装方針
1. `src/app/page.tsx` (想定) でトップページを構築し、セクションごとにコンポーネント化。
2. UIライブラリは既存スタック (Next.js + CSS Modules/Tailwind 等) に合わせる。グラデーション/グローは CSS で再現。
3. データは一旦静的モックとして `src/data/topCasts.ts` を作成し、型定義で保守性を確保。
4. 画像はダミーを使用。最終的な差し替えに備え `public/images/top-casts/` 配下を前提にする。

## 4. タスク分解
1. **情報設計**: セクション構造・コンポーネント階層案を決定。
2. **データスキーマ作成**: キャスト情報の型・モックデータ準備。
3. **レイアウト実装**
   - ヘッダー
   - CTA/広告エリア
   - トップキャストグリッド
   - 広告(BOTTOM)
   - フッター
4. **スタイル適用**
   - カラーパレット/フォント設定
   - グラデーション/グローエフェクト
   - レスポンシブ調整 (SP/タブレット/PC)
5. **リンク挙動設定**: クリック可能にする (遷移先は仮の `#` または `router.push` のみ)。
6. **テスト/確認**: ビジュアル確認、Lighthouse で簡易チェック、Lint/Type Check。

## 5. スケジュール (目安)
- Day 1: 要件再確認、情報設計、モックデータ設計。
- Day 2: レイアウト実装 (ヘッダー〜キャストカード)。
- Day 3: 広告エリア/フッター実装、スタイル調整。
- Day 4: レスポンシブ最適化、ホバーアニメーション、微調整。
- Day 5: テスト、コードレビュー、成果物共有。

## 6. リスクと対策
- **ビジュアル要件の解釈差異**: 早期にデザインモックを簡易作成し確認。
- **ダミー画像調達**: 一時的にプレースホルダーサービス使用、リリース前に差し替えタスク設定。
- **パフォーマンス**: 画像最適化 (`next/image`) を使用しつつレイアウトシフトに注意。
